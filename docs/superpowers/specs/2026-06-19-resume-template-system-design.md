# Resume Template System 设计文档

## 1. 概述

当前项目已经从单一校招简历编辑器，扩展为面向学生、职场人和通用求职者的统一简历编辑器内核。

下一步不做 Canva 式自由设计器，也不做大量模板市场，而是在现有编辑器和打印链路之上，增加一套轻量、多模板、可稳定导出的简历模板系统。

本次设计目标是：

- 让产品看起来更像真正的简历工具，而不只是单一白底表单预览
- 保持一套统一编辑器内核，不拆分多套编辑器
- 让用户填写一份内容后，可以在多个模板之间切换对比
- 让实时预览和最终 PDF 输出保持一致

## 2. 产品目标与边界

### 2.1 产品目标

- 在现有统一编辑器内核上新增多模板能力
- 提供至少 3 个可用、风格明确的正式简历模板
- 让模板切换立即反映到右侧实时预览
- 让打印 / PDF 始终跟随当前选中的模板输出
- 保持用户当前模块内容、身份体系、导入导出和本地保存能力不变

### 2.2 非目标

- 不做自由布局拖拽排版
- 不做十几个以上的模板市场
- 不让模板改变左侧编辑器结构
- 不让模板引入不同字段模型或不同模块规则
- 不做模板商城、付费体系或模板下载中心
- 不做每个模板一套完全不同的产品页面

### 2.3 首版原则

- 一个编辑器内核
- 一份内容，多模板切换
- 模板只影响预览和打印，不影响编辑数据结构
- 首版优先保证打印质量，而不是模板数量

## 3. 首版模板集合

首版固定提供 3 个模板：

- `经典简历`
- `双栏简历`
- `校招简历`

三者都属于正式求职简历模板，不做过度花哨的设计表达。

### 3.1 经典简历

- 单栏结构
- 强调专业、稳重、清晰
- 适合职场人和通用求职场景

### 3.2 双栏简历

- 左右分栏结构
- 左栏承载短信息模块
- 右栏承载主体经历模块
- 视觉上更接近成熟简历产品中的现代模板

### 3.3 校招简历

- 单栏结构
- 视觉更轻、更清爽
- 更突出教育、项目、校园经历等学生常见重点

## 4. 身份与模板的关系

身份和模板不强绑定。

### 4.1 已确认规则

- 学生、职场人、通用求职者都可以自由切换任意模板
- 系统只为每种身份提供一个默认推荐模板
- 用户手动选择过模板后，后续身份切换不应偷偷覆盖该选择

### 4.2 推荐关系

首版建议默认推荐如下：

- 学生：`校招简历`
- 职场人：`经典简历`
- 通用求职者：`经典简历`

`双栏简历` 作为可自由切换的增强模板，不作为任何身份的唯一默认模板。

## 5. 数据模型与持久化

模板选择是简历状态的一部分，而不是临时界面状态。

### 5.1 状态新增字段

在当前简历存储状态中新增：

- `templateId`
- `hasUserSelectedTemplate`

语义如下：

- `templateId`：当前简历正在使用的模板
- `hasUserSelectedTemplate`：用户是否主动切换过模板

### 5.2 持久化规则

- `templateId` 写入 `localStorage`
- `hasUserSelectedTemplate` 写入 `localStorage`
- JSON 备份导出时包含模板相关字段
- JSON 导入时恢复模板相关字段

### 5.3 旧数据迁移

旧本地数据和旧 JSON 备份没有模板字段时，按以下规则自动补齐：

- 如果存在已保存身份，则按身份补默认模板
- 如果身份缺失但能迁移到新结构，则按迁移后的身份补默认模板
- `hasUserSelectedTemplate` 默认记为 `false`

这样可以保证老数据升级后立即可用，同时不给用户造成“模板被我选过”的错误感知。

## 6. 模板注册表

模板能力通过集中注册表管理，不在多个组件中散落硬编码。

每个模板至少声明：

- `id`
- `name`
- `description`
- `defaultForIdentities`
- `previewRenderer`
- `thumbnailRenderer`
- `printClassName`

注册表的目标是：

- 集中管理模板元数据
- 统一模板切换和默认映射逻辑
- 控制不同模板的预览和打印样式入口

## 7. 界面与交互

### 7.1 模板选择器位置

模板选择器固定放在右侧 `实时预览` 标题区下方，常驻显示。

不放顶部操作栏，也不做弹层或抽屉。

### 7.2 模板选择器形态

首版采用完整缩略图选择器，而不是下拉菜单。

模板选择器由 3 张模板卡组成，每张卡包含：

- 模板实时缩略图
- 模板名称
- 一句简短说明
- 当前选中状态高亮

### 7.3 缩略图策略

首版使用当前简历内容生成实时缩略图，而不是固定风格占位图。

这样用户能直接对比：

- 同一份真实内容在不同模板中的排版差异
- 哪个模板更适合当前简历

### 7.4 切换行为

用户点击某张模板卡后：

- 右侧主预览立即切换为目标模板
- 当前模板写入状态和本地存储
- 后续打印 / PDF 导出跟随该模板

切换模板时：

- 不改左侧编辑结构
- 不改模块内容
- 不自动增删模块
- 不自动改模块标题

## 8. 渲染架构

模板系统采用“统一数据 + 分模板渲染外壳”的方式实现。

### 8.1 保持共享的部分

以下能力继续共享，不按模板重复实现：

- 模块数据结构
- 模块编辑逻辑
- 模块排序逻辑
- 模块内容基础 renderer
- 身份分流与推荐配置
- 本地存储与 JSON 备份

### 8.2 模板负责的部分

模板主要负责：

- 页面骨架
- 标题层级
- 分栏策略
- 模块间距与留白
- 分隔线、边框、背景和局部强调
- 打印态样式入口

### 8.3 组件边界

首版建议拆分为：

- `TemplateSelector`
- `ResumeTemplateRenderer`
- `ClassicResumeTemplate`
- `SidebarResumeTemplate`
- `CampusResumeTemplate`

其中：

- `TemplateSelector` 负责模板卡列表与切换
- `ResumeTemplateRenderer` 根据 `templateId` 选择模板组件
- 具体模板组件负责布局和样式壳子

## 9. 模块渲染与模板差异

### 9.1 经典简历

- 沿用单栏阅读顺序
- 模块按当前顺序渲染
- 使用更成熟、正式的标题和分隔规则

### 9.2 校招简历

- 仍为单栏
- 模块按当前顺序渲染
- 视觉上更轻，教育、项目、校园经历更突出

### 9.3 双栏简历

双栏模板由模板自己决定左右栏模块分配规则，不做“按当前顺序自动切两半”。

首版建议分配原则：

- 左栏优先放：个人信息、技能、证书、核心能力、个人亮点
- 右栏优先放：职业摘要、教育经历、经历、项目经历、校园经历、奖项荣誉

如果某模板推荐栏位中缺少对应模块，则：

- 只消费现有模块
- 自然留空或收紧版面
- 不自动补模块

## 10. 打印与 PDF

打印和 PDF 是模板系统的强约束。

### 10.1 输出一致性

- 当前主预览是什么模板
- 最终打印 / PDF 就输出什么模板

不允许出现：

- 预览是一个模板
- 导出又回退到默认模板

### 10.2 打印边界

模板系统不能破坏当前打印链路：

- 继续隐藏编辑区与页面辅助元素
- 保持 A4 简历主体稳定输出
- 多页内容允许自然分页

### 10.3 风险点

双栏模板比单栏模板更容易出现：

- 分页细节不稳定
- 左右栏高度不平衡
- 某些模块断页体验变差

因此首版打印验收重点应放在双栏模板。

## 11. 默认模板行为

### 11.1 首次进入

当用户第一次进入某个身份时：

- 系统按身份分配默认模板
- 此时 `hasUserSelectedTemplate = false`

### 11.2 用户主动切换后

当用户主动切换模板后：

- 更新 `templateId`
- 设置 `hasUserSelectedTemplate = true`

### 11.3 后续切换身份

后续如果用户切换身份：

- 若 `hasUserSelectedTemplate = false`，允许跟随新身份更新默认模板
- 若 `hasUserSelectedTemplate = true`，保留用户已选模板，不自动覆盖

这样可以兼顾：

- 系统默认引导
- 用户自主选择

## 12. 测试要求

首版至少覆盖以下验证：

### 12.1 状态与迁移

- 老本地数据进入后会自动补默认模板
- 老 JSON 备份导入后会自动补默认模板
- 新 JSON 导出包含 `templateId`
- 新 JSON 导入能恢复模板
- `hasUserSelectedTemplate` 行为符合预期

### 12.2 交互

- 模板选择器在预览标题区下方常驻显示
- 点击模板卡后主预览立即切换
- 当前模板卡有选中高亮
- 已保存模板刷新后仍正确显示

### 12.3 模板行为

- `经典简历` 正常渲染
- `校招简历` 正常渲染
- `双栏简历` 的左右栏分配规则生效
- 缺失某些模块时模板仍稳定显示

### 12.4 打印与回归

- 打印 / PDF 跟随当前模板
- 原有身份切换、导入导出、模块排序、照片显示不被破坏
- `npm test` 通过
- `npm run build` 通过

## 13. 风险与实现控制

### 13.1 主要风险

- 实时缩略图会带来额外渲染开销
- 双栏模板的打印分页复杂度更高
- 当前样式从单模板演进而来，模板化时需要控制 CSS 污染

### 13.2 实现控制原则

- 首版只做 3 个模板
- 不同时推进更多模板或自由布局
- 优先保证实时预览与打印一致
- 优先保证模板切换稳定，再做视觉细节打磨

## 14. 结论

本次模板系统不是把产品升级为自由设计工具，而是在现有统一编辑器基础上，增加一层轻量但真实可用的模板能力。

首版的成功标准不是“模板很多”，而是：

- `经典简历 / 双栏简历 / 校招简历` 三个模板真实可用
- 用户能基于同一份内容即时切换模板
- 预览和 PDF 一致
- 旧数据可平滑迁移

只要这四点成立，产品的完成度和专业感都会明显上一个台阶。
