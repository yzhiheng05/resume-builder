# Canva Style Resume Editor 设计文档

## 1. 概述

当前项目已经具备通用求职简历编辑器的基础能力：

- 身份预设
- 官方模块库
- 模块实例数据模型
- 实时 A4 预览
- 多模板切换
- 模块拖拽排序
- 本地保存
- JSON 导入 / 导出
- 浏览器打印 / PDF 导出

本次设计目标是把产品从“表单 + 预览”的工具形态，升级为更接近 Canva 和主流在线简历网站的网页版外部端编辑器。

首版不做完整自由设计器，而是做“Canva 感的结构化简历编辑器”：用户可以从模块库添加内容，在 A4 画布中选中和排序模块，在属性面板中编辑内容与样式，并稳定导出正式求职简历 PDF。

## 2. 目标与边界

### 2.1 产品目标

- 提供更产品化、更像主流简历网站的编辑体验
- 让用户能直观看到“模块库、画布、属性设置”的编辑结构
- 支持多种模块添加、复制、删除、显隐和排序
- 支持选中画布模块后集中编辑
- 支持基础样式控制，让同一份内容可以呈现不同视觉风格
- 保持当前本地保存、JSON 备份、多模板和 PDF 导出能力
- 保持正式求职简历的结构稳定性和打印可靠性

### 2.2 非目标

- 不做任意坐标的自由拖拽布局
- 不做图层面板、素材市场、贴纸、装饰元素和自由图形编辑
- 不做多人协作、账号系统、云端同步、模板商城和付费体系
- 不让模板或样式设置改变底层简历内容结构
- 不把产品扩展成通用海报、作品集或个人主页设计器
- 不在首版实现完整移动端编辑体验

### 2.3 核心原则

- 结构化内容优先，画布体验增强
- 模板和样式只影响预览与打印
- 模块数据仍由统一编辑器内核管理
- PDF 导出稳定性优先于自由设计能力
- 首版只做用户能真实感知的编辑体验升级

## 3. 用户体验设计

### 3.1 主界面结构

编辑器主界面采用三栏结构：

- 左栏：模块库
- 中间：A4 简历画布
- 右栏：属性面板

顶部保留产品级操作区，用于身份切换、导入、导出、重置和打印 / PDF 导出。

### 3.2 左栏模块库

模块库展示官方支持的简历模块。首版模块包括：

- 个人信息
- 职业摘要
- 教育经历
- 工作 / 实习经历
- 项目经历
- 技能
- 证书
- 奖项荣誉
- 校园经历
- 核心能力
- 个人亮点

交互规则：

- 点击模块即可添加到当前简历
- 支持多实例的模块可以重复添加
- 不支持多实例的模块只允许存在一个，例如个人信息
- 已添加但可重复的模块仍可继续添加
- 模块库显示模块用途说明，帮助用户判断该加什么

### 3.3 中间 A4 画布

画布是用户主要感知到的“Canva 感”区域。

画布能力：

- 显示当前模板下的 A4 简历预览
- 模块按当前顺序渲染
- 点击模块后进入选中状态
- 选中模块显示清晰边框或高亮
- 模块可以在画布中拖拽排序
- 当前选中模块与右侧属性面板同步
- 空状态提示用户从左侧添加模块

约束：

- 模块只能按文档流排序，不允许拖到任意坐标
- 双栏模板内部仍由模板规则决定左右栏归属
- 画布交互不影响打印时的纯简历输出

### 3.4 右栏属性面板

属性面板负责集中编辑当前选中对象。

当选中模块时，显示：

- 模块标题
- 模块显隐
- 模块内容编辑表单
- 复制模块
- 删除模块
- 根据模块类型展示的条目编辑能力

当未选中模块时，显示全局设置：

- 当前模板
- 主题色
- 字体密度
- 模块间距
- 标题样式

属性面板不承担模块库职责，避免左栏和右栏功能混乱。

## 4. 功能范围

### 4.1 模块操作

首版支持：

- 添加模块
- 删除模块
- 复制模块
- 开启 / 隐藏模块
- 修改模块标题
- 画布中拖拽排序
- 点击画布模块后选中
- 删除当前选中模块后自动选择相邻模块或回到全局设置

删除规则：

- 删除前需要二次确认
- 个人信息模块如果是唯一实例，默认不允许删除，只允许隐藏可选字段
- 删除模块会同时从模块列表和排序中移除

复制规则：

- 仅支持可多实例模块复制
- 复制后保留内容副本，但生成新的模块 id
- 复制后的模块插入到原模块后方并自动选中

### 4.2 内容编辑

内容编辑继续复用当前模块形态：

- `personal`：个人信息
- `text`：职业摘要等纯文本模块
- `timeline`：教育、工作 / 实习、项目、校园经历
- `list`：技能、证书、奖项、核心能力、个人亮点

首版不引入任意自定义字段类型。

### 4.3 模板与样式

模板继续使用现有模板系统：

- 经典简历
- 双栏简历
- 校招简历

样式设置新增为轻量全局配置：

- 主题色
- 字体密度
- 模块间距
- 标题样式

样式只影响预览和打印，不改变模块数据。

### 4.4 本地保存与导入导出

以下状态需要持久化：

- 模块数据
- 模块顺序
- 当前选中身份
- 当前模板
- 样式设置
- JSON 备份中的新增样式字段

旧数据迁移规则：

- 没有样式字段的旧数据自动补默认样式
- 旧 JSON 导入后也自动补默认样式
- 旧模块数据不被重排、不被删除

## 5. 数据模型

### 5.1 状态新增

在现有 Zustand 状态中增加：

- `activeModuleId`
- `resumeStyle`

`activeModuleId` 表示当前选中的模块。为空时，右侧属性面板显示全局设置。

`resumeStyle` 建议结构：

```ts
interface ResumeStyleSettings {
  accentColor: string;
  density: "compact" | "comfortable";
  sectionSpacing: "tight" | "normal" | "loose";
  headingStyle: "underline" | "bar" | "plain";
}
```

### 5.2 模块动作

状态层新增或整理以下动作：

- `selectModule(moduleId: string | null)`
- `addModule(kind: ModuleKind)`
- `duplicateModule(moduleId: string)`
- `deleteModule(moduleId: string)`
- `toggleModuleVisible(moduleId: string)`
- `updateModuleTitle(moduleId: string, title: string)`
- `updateResumeStyle(nextStyle: Partial<ResumeStyleSettings>)`

这些动作都由 store 统一处理，组件不直接拼装模块结构。

## 6. 组件架构

### 6.1 新增组件

建议新增：

- `ModuleLibraryPanel`
- `InspectorPanel`
- `ModuleInspector`
- `GlobalStyleInspector`
- `CanvasShell`

### 6.2 调整组件

建议调整：

- `App`
  - 从两栏布局调整为三栏编辑器壳
  - 负责串联模块库、画布和属性面板
- `PreviewPanel`
  - 强化画布选中态
  - 保持拖拽排序能力
  - 将模板选择入口迁移或同步到属性面板的全局设置区
- `ResumeTemplateRenderer`
  - 接收 `resumeStyle`
  - 将样式设置传递到具体模板
- `TemplateSelector`
  - 可复用于全局属性面板

### 6.3 保持不变

以下部分保持当前思路：

- 模块注册表
- 模块内容数据结构
- 身份预设
- 模板注册表
- JSON 备份机制
- 打印入口

## 7. 数据流

1. 用户从左栏点击添加模块
2. `addModule` 在 store 中创建模块实例并更新顺序
3. 新模块自动成为 `activeModuleId`
4. 中间画布按模块顺序实时渲染
5. 右侧属性面板读取 `activeModuleId` 并展示对应编辑器
6. 用户编辑内容或样式后写回 store
7. store 持久化到 localStorage
8. 导出 JSON 时带出模块、模板和样式设置
9. 打印 / PDF 导出读取当前模板和样式渲染纯简历内容

## 8. 错误与边界处理

- 如果 `activeModuleId` 指向不存在的模块，自动清空选中态
- 如果用户删除当前选中模块，优先选中同位置后一个模块；没有则选中前一个；仍没有则进入全局设置
- 如果旧数据缺少 `resumeStyle`，使用默认样式
- 如果 JSON 导入缺少样式字段，导入成功但补默认样式
- 如果 JSON 导入样式字段非法，忽略非法字段并保留可用内容
- 如果用户尝试添加不可重复模块且已存在，选中已有模块而不是重复创建
- 打印模式必须隐藏模块库、属性面板、顶部操作区和所有画布交互边框

## 9. 验收标准

- 用户能从左侧模块库添加多种简历模块
- 用户能添加可重复模块的多个实例
- 用户点击画布模块后，该模块进入选中态并在右侧显示编辑表单
- 用户能在右侧属性面板修改模块标题、内容、显隐
- 用户能复制和删除可多实例模块
- 用户能在画布中拖拽模块排序
- 用户能切换模板和基础样式，预览实时更新
- 刷新页面后内容、模块顺序、模板和样式设置保留
- JSON 导出 / 导入能保留新增样式设置
- 旧本地数据和旧 JSON 数据能自动补默认样式
- 浏览器打印 / PDF 导出不包含编辑器 UI
- 打印内容无明显重叠、裁切或交互边框残留
- 桌面端体验完整，移动端至少可查看预览和执行基础操作

## 10. 验证计划

### 10.1 自动验证

- `npm test`
- `npm run build`

测试重点：

- store 新增动作
- 样式字段默认值和迁移
- JSON 备份兼容
- 模块添加、复制、删除和选中态
- 画布排序回归
- 渲染入口能接收模板和样式

### 10.2 浏览器验证

手工检查：

- 首屏三栏布局
- 模块库添加模块
- 画布选中模块
- 右侧属性面板编辑内容
- 模板切换
- 样式切换
- 刷新后持久化
- 导出 PDF 前后编辑器 UI 隐藏

### 10.3 打印验证

至少检查：

- 默认内容打印
- 长内容多页打印
- 双栏模板打印
- 样式设置后的打印

重点确认：

- 无编辑器 UI 进入 PDF
- 无选中边框进入 PDF
- 无模块库 / 属性面板进入 PDF
- 内容不明显重叠或裁切
