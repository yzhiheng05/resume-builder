# Campus Resume Builder MVP 设计文档

## 1. 概述

本项目目标是做一个桌面端优先的网页版校招简历制作工具。用户在左侧按模块填写简历内容，右侧实时看到 A4 比例的简历预览，并可直接在预览区拖动模块顺序，最终通过浏览器打印导出 PDF。

第一版聚焦“好填、好看、好导出”，不做设计工具级自由排版，也不引入账号、云同步和多模板系统。

## 2. 目标用户

- 主要用户：学生 / 校招求职者
- 核心诉求：
  - 不想手动折腾 Word 排版
  - 想边写边看简历样式
  - 想快速调整各模块先后顺序
  - 想稳定导出为 PDF 投递

## 3. 产品目标

- 提供结构化、低门槛的校招简历填写体验
- 提供实时预览，减少“写完才发现排版难看”的问题
- 提供模块级排序能力，让用户快速调整简历重点
- 提供稳定的本地保存和 PDF 导出能力

## 4. 非目标

- 不实现任意位置自由拖拽布局
- 不实现多模板切换
- 不实现账号体系和云端同步
- 不实现导出图片
- 不实现自定义模块类型
- 不实现完整移动端编辑体验

## 5. 产品形态

第一版采用单页双栏编辑器形态：

- 顶部：基础操作栏
- 左侧：模块化编辑区
- 右侧：A4 比例简历预览区

用户在左侧填写内容，右侧实时更新。用户可在预览区直接拖动整个模块的上下顺序，排序结果同步回编辑区。导出时通过打印样式隐藏编辑区，只保留简历主体。

## 6. 信息模块

第一版采用固定模块 + 可开关显示的方案。默认模块包括：

- 个人信息
- 教育经历
- 项目经历
- 实习经历
- 校园经历
- 技能
- 获奖 / 证书

规则如下：

- 模块类型固定，用户不能新增全新的模块类别
- 每个模块可单独显示或隐藏
- 对于经历类模块，允许新增多条条目
- 模块顺序可调整，但模块内部字段结构保持固定
- 个人信息模块默认位于最上方，但允许用户像其他模块一样调整顺序

## 7. 核心交互

### 7.1 编辑区

左侧编辑区采用模块卡片列表：

- 每张卡片包含模块名称、显示开关、展开 / 收起控制
- 展开后展示该模块的表单字段
- 对于可重复条目，提供新增和删除操作

目标是让用户按常规填表方式完成编辑，而不是学习复杂排版逻辑。

### 7.2 预览区

右侧预览区渲染为 A4 比例的简历画布：

- 实时反映左侧表单内容
- 呈现统一样式的单模板简历
- 支持模块级拖动排序

个人信息模块也参与排序，只是默认排在最前。第一版仍然只支持整块模块上下排序，不支持模块内部自由布局。

拖动范围仅限整块模块上下排序，不支持把模块拖到任意坐标位置，也不支持重叠、吸附或自由布局。这样可以保证简历视觉稳定，也能降低 PDF 导出风险。

### 7.3 本地保存

第一版采用浏览器本地保存：

- 用户输入时自动保存
- 刷新页面后恢复上次编辑状态
- 不引入登录和后端存储

### 7.4 导出

第一版仅支持浏览器打印 / 导出 PDF：

- 打印时隐藏编辑区和非必要操作 UI
- 保留 A4 简历主体
- 通过打印样式控制页边距、字体、分隔线和分页表现
- 当内容超出一页时，允许自动分页为多页 A4，而不是强行压缩到单页

## 8. 页面结构

页面结构保持最小必要复杂度：

- 顶部操作栏
  - 新建 / 重置
  - 本地保存状态提示
  - 导出 PDF
- 主体双栏
  - 左栏：模块编辑区
  - 右栏：简历预览区

桌面端优先保证双栏体验。手机端第一版只要求基础可查看，不承诺完整编辑效率。

## 9. 数据结构建议

第一版数据结构建议分为三层：

- `resume`：整份简历的根对象
- `sections`：各模块内容数据
- `sectionOrder`：模块显示顺序

另外，每个模块还需要单独记录显示状态。核心思想是把“内容数据”和“排序信息”分开存储，这样拖动排序时只改顺序数组，不影响模块内容结构。

由于个人信息模块也可排序，所以它和其他模块一样由 `sectionOrder` 控制，只是在默认数据里排第一。

## 10. 技术方向

第一版采用纯前端单页应用实现，技术上保持轻量：

- 使用 React 生态实现交互界面
- 使用轻量状态方案管理简历数据、显示状态和排序信息
- 使用成熟拖拽库实现模块排序
- 使用 CSS 打印样式处理 PDF 导出

明确不采用以下复杂方案：

- 富文本编辑器内核
- 画布式自由排版引擎
- 服务端 PDF 渲染
- 数据库和鉴权系统

## 11. 成功标准

满足以下条件即可视为 MVP 达标：

1. 用户能在桌面端顺畅填写校招简历常见模块
2. 用户能开关模块显示
3. 用户能在预览区拖动模块顺序，且编辑区顺序同步
4. 右侧预览能实时反映左侧修改
5. 刷新页面后，本地内容仍然保留
6. 用户能通过浏览器打印稳定导出为 PDF
7. 当内容超过一页时，导出结果允许自然分页，不出现明显错位、重叠或内容裁切
8. 手机端至少可以查看页面内容

## 12. 验证方式

实现完成后，至少执行以下验证：

1. 手工填写一份完整示例简历，检查所有模块显示和编辑正常
2. 在预览区拖动多个模块，确认编辑区顺序同步且无异常
3. 刷新页面，确认本地数据能正确恢复
4. 打印预览时确认只显示简历主体，没有编辑面板
5. 使用超出单页的示例数据测试，确认 PDF 能自然分页且无明显裁切、重叠、错位

## 13. 风险与后续边界

- 打印导出效果受浏览器打印能力影响，必须在实现阶段重点验证
- 如果后续要支持多模板，需要提前考虑样式层与数据层解耦
- 如果后续要支持自由布局，现有排序型结构将不足，需要升级为更重的版面模型

当前版本不提前为这些扩展做复杂抽象，只以支撑已确认 MVP 为准。
