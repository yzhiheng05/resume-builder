# 项目需求

## 1. 背景

- 要解决的问题：为学生和校招求职者提供一个更美观、更直观的网页版简历制作工具，降低排版门槛。
- 业务背景：用户希望在填写简历信息时实时看到排版效果，并能通过拖动模块顺序快速调整简历结构。

## 2. 目标

- 核心目标：实现一个桌面端优先的简历编辑器 MVP，支持模块化填写、实时预览、模块开关、预览区拖动排序和浏览器打印导出 PDF。
- 非目标：不在第一版实现账号系统、云端同步、多模板切换、自由布局拖拽、自定义模块类型和完整移动端编辑体验。

## 3. 范围

- 本次包含：
  - 校招常见模块的结构化填写
  - 固定模块的显示开关
  - 预览区模块级拖动排序
  - 浏览器本地保存
  - A4 预览与打印导出 PDF
  - 内容超出一页时的自动多页打印
- 本次不包含：
  - 任意位置自由拖拽排版
  - 登录、注册、云端保存
  - 多模板切换
  - 导出图片
  - 手机端完整编辑交互

## 4. 约束

- 技术约束：第一版采用纯前端实现，不依赖后端服务。
- 时间约束：优先做最小可用版本，避免进入设计工具级复杂度。
- 其他约束：桌面端优先，手机端只要求基础可查看。

## 5. 验收标准

- [ ] 用户能填写个人信息、教育经历、项目经历、实习经历、校园经历、技能、获奖/证书等模块
- [ ] 用户能开关模块显示
- [ ] 用户能在预览区拖动模块顺序，且编辑区与预览区顺序保持一致
- [ ] 个人信息模块默认在最上方，但也可以参与拖动排序
- [ ] 预览区能实时反映编辑内容
- [ ] 刷新页面后，本地数据仍然保留
- [ ] 用户能通过浏览器打印稳定导出 PDF，导出内容无明显错位、重叠；内容超出一页时允许自然分页
